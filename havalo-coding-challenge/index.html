Hello, candidate!

If you're reading this in your web-browser, that means you have successfully compiled and started the Havalo coding challenge web-server!

This coding exercise is a two part challenge that demonstrates proficiency in fundamental Java, JavaScript, and basic web-application principles. You will be writing Java code, and JavaScript (using jQuery). Your JavaScript will be making AJAX calls from the browser to the local web-server running at http://localhost:4444.

There are 2 files in this exercise:

  1. HavaloCodingChallenge.java
  2. words.html

You will be editing both of these files.

The exercise is fairly simple, but requires a little creativity on your part.

You will be writing and wiring together a little web-app that takes in word (a String) and does the following:

  • Displays whether or not the input word a palindrome.
  • Displays whether or not the input word contains duplicate characters.
  • The input word, in reverse.

At the top of HavaloCodingChallenge.java you'll find 3 empty Java methods that you are required to fill in:

  • public static boolean isPalindrome(String word)
  • public static boolean containsDuplicateCharacters(String word)
  • public static String reverseWord(String word)

Once you've got the Java part figured out, check out words.html. Inside of words.html, you'll be writing JavaScript (using jQuery) that will make AJAX requests to the tiny-web server on http://localhost:4444. The AJAX requests, when received by the web-server, will invoke your 3 Java methods and will return the result to the browser. It is your task/challenge to write code in words.html that calls the server from JavaScript and displays the result — that is, display if the input word is a palindrome, if the input word contains duplicate characters, and the input word in reverse. How you accept the "input word" from the user and display the results from the server is totally your choice. Make it look pretty, wow us!

You are required to use jQuery to make the AJAX requests from the browser, but are allowed to use any additional JavaScript libraries and CSS toolkits of your choice if you'd like to make words.html look beautiful.
